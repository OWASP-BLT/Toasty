# Contributing to Toasty

Thank you for your interest in contributing to Toasty! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/Toasty.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes locally
6. Commit your changes: `git commit -m "Description of changes"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Development Setup

Follow the setup instructions in the [README.md](README.md) to get your local development environment running.

## Code Style

- Use clear, descriptive variable and function names
- Add comments for complex logic
- Follow existing code patterns in the repository
- Keep functions focused and single-purpose

## Testing

Before submitting a PR:

1. Test your changes locally
2. Verify the app responds correctly to `/plan` commands
3. Check for any console errors or warnings

## Pull Request Process

1. Update the README.md with details of changes if applicable
2. Ensure your PR description clearly describes the problem and solution
3. Link any relevant issues in the PR description
4. Be responsive to feedback and review comments

## Questions?

If you have questions, feel free to open an issue or ask in your PR.
